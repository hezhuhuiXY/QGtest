# token

- 身份认证：登录成功后服务器发 token，前端后续请求都带上它
- 访问控制：服务器根据 token 里的用户信息/角色，判断能否访问接口
- 免重复登录：在 token 有效期内，不用每次都输账号密码
- 无状态会话：服务端不一定要存传统 session，扩展更方便
- 安全边界：token 过期、签名错误、被篡改时可直接拒绝请求
